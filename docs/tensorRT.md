# TensorRT Build and Test Tutorial

This tutorial explains two TensorRT deployment paths and how to evaluate AP:

- Path A: PyTorch -> ONNX -> TensorRT engine
- Path B: Direct PyTorch -> TensorRT (Torch-TensorRT)

For the consolidated AP measurement workflow and result locations, see `docs/apEvaluation.md`.

## Prerequisites

- TensorRT installed, including trtexec.
- Python bindings for TensorRT and pycuda available in the environment.
- ONNX model already exported.
- For direct path: Torch-TensorRT installed in the same Python environment.

Example direct path install (version must match your CUDA/PyTorch stack):

```bash
pip install torch-tensorrt
```

## 1. Build TensorRT engine from ONNX

```bash
python3 opencood/tools/build_tensorrt_engine.py \
  --onnx_model <MODEL_DIR>/pipeline_a/model.onnx
```

When `--engine_path` is omitted, the builder writes the engine next to the ONNX file
using the same basename with `.engine`, for example:

- ONNX: `<MODEL_DIR>/pipeline_a/model.onnx`
- Engine: `<MODEL_DIR>/pipeline_a/model.engine`

If `<MODEL_DIR>/pipeline_a/model.onnx.meta.yaml` exists (generated by `export_onnx.py`),
the engine build automatically uses `suggested_trt_profile` and you do not need to
provide `--min_shapes/--opt_shapes/--max_shapes` manually.

Important deployment note:

- Dynamic TensorRT engines still need bounded min/opt/max profiles.
- Runtime shapes outside `max_shapes` will fail at `setInputShape`.
- For intermediate fusion, the most common limiter is voxel count.

Easy way to increase deployment headroom:

```bash
python3 opencood/tools/build_tensorrt_engine.py \
  --onnx_model <MODEL_DIR>/pipeline_a/model.onnx \
  --max_voxels_margin 2
```

`--max_voxels_margin` multiplies detected max voxel profile count (from metadata/profile).

Absolute override is also available:

```bash
python3 opencood/tools/build_tensorrt_engine.py \
  --onnx_model <MODEL_DIR>/pipeline_a/model.onnx \
  --max_voxels 30000
```

You can combine both flags. The builder takes the larger value between:

- margin-scaled max profile count
- absolute `--max_voxels`

The builder always prints:

- detected max voxel profile count
- final max voxel profile count per voxel input

`--max_voxel_margin` (singular) is accepted as an alias of `--max_voxels_margin`.

Optional explicit metadata path:

```bash
python3 opencood/tools/build_tensorrt_engine.py \
  --onnx_model <MODEL_DIR>/pipeline_a/model.onnx \
  --engine_path <MODEL_DIR>/pipeline_a/model_fp32.engine \
  --profile_yaml <MODEL_DIR>/pipeline_a/model.onnx.meta.yaml
```

Optional FP16 build:

```bash
python3 opencood/tools/build_tensorrt_engine.py \
  --onnx_model <MODEL_DIR>/pipeline_a/model.onnx \
  --engine_path <MODEL_DIR>/pipeline_a/model_fp16.engine \
  --fp16
```

Optional shape profile example:

```bash
python3 opencood/tools/build_tensorrt_engine.py \
  --onnx_model <MODEL_DIR>/pipeline_a/model.onnx \
  --engine_path <MODEL_DIR>/pipeline_a/model_fp16.engine \
  --fp16 \
  --min_shapes "voxel_features:1x32x4,voxel_coords:1x4,voxel_num_points:1" \
  --opt_shapes "voxel_features:12000x32x4,voxel_coords:12000x4,voxel_num_points:12000" \
  --max_shapes "voxel_features:20000x32x4,voxel_coords:20000x4,voxel_num_points:20000"
```

Important: when ONNX inputs are dynamic, build now fails early if no valid shape profile is found.

## 2. Run TensorRT inference and AP evaluation

```bash
python3 opencood/tools/inference_tensorrt.py \
  --model_dir <MODEL_DIR> \
  --engine_path <MODEL_DIR>/pipeline_a/model_fp32.engine \
  --fusion_method <early|late|intermediate> \
  --output_yaml pipeline_a/eval_tensorrt.yaml \
  --test
```

Output:

- AP metrics are saved to <MODEL_DIR>/pipeline_a/eval_tensorrt.yaml.

Runtime behavior notes:

- The TensorRT runner supports both legacy bindings API and TensorRT 10 I/O tensor API.
- Input tensors are cast to the engine-expected dtype before execution to prevent dtype mismatch failures.
- If TensorRT reports unresolved output shapes, the current sample is outside the engine profile range.

## 3. Build direct PyTorch -> TensorRT artifact (Torch-TensorRT)

```bash
python3 opencood/tools/build_tensorrt_engine_direct.py \
  --model_dir <MODEL_DIR> \
  --fusion_method <early|late|intermediate> \
  --output <MODEL_DIR>/pipeline_b/model_trt.ts \
  --dynamic_shapes \
  --max_search_batches 200 \
  --max_voxels_margin 2 \
  --precision fp32
```

Direct Torch-TensorRT build is now compatible with dynamic input shapes.

- Enable it with `--dynamic_shapes`.
- If `<MODEL_DIR>/pipeline_a/model.onnx.meta.yaml` exists, direct build now reuses its profile hints and skips full Stage 3 scan.
- You can also pass `--profile_yaml <PATH_TO_META_YAML>` explicitly.
- `--max_search_batches` limits Stage 3 profile scanning. Use this to avoid very long full-dataset scans.
- `--scan_log_interval` controls periodic Stage 3 progress logs (default: 50 batches).
- `--max_voxels_margin` and `--max_voxels` are supported with the same semantics as `build_tensorrt_engine.py`.
- The builder prints detected and final max voxel profile counts.

Optional FP16 direct compile:

```bash
python3 opencood/tools/build_tensorrt_engine_direct.py \
  --model_dir <MODEL_DIR> \
  --fusion_method <early|late|intermediate> \
  --output <MODEL_DIR>/pipeline_b/model_trt_fp16.ts \
  --dynamic_shapes \
  --max_search_batches 200 \
  --max_voxels_margin 2 \
  --precision fp16
```

This command also writes metadata at `<artifact>.meta.yaml` with input order, precision, trace input shapes/dtypes, and suggested dynamic profile ranges.

## 4. Run AP evaluation for direct TensorRT artifact

```bash
python3 opencood/tools/inference_tensorrt_direct.py \
  --model_dir <MODEL_DIR> \
  --trt_module <MODEL_DIR>/pipeline_b/model_trt.ts \
  --fusion_method <early|late|intermediate> \
  --output_yaml pipeline_b/eval_tensorrt_direct.yaml \
  --test
```

Output:

- AP metrics are saved to <MODEL_DIR>/pipeline_b/eval_tensorrt_direct.yaml.

## Troubleshooting

- Python import error for pycuda or tensorrt:
  Confirm the active environment matches the TensorRT installation.
- Engine build fails:
  Try FP32 first, then FP16. For dynamic ONNX models, ensure metadata exists or pass explicit shapes.
- Dynamic shape runtime errors:
  Rebuild engine with wider max shape profile (`--max_shapes`), or use `--max_voxels` for the common voxel-limit case, or regenerate ONNX metadata from a richer trace batch.
- torch_tensorrt import/compile error:
  Verify Torch-TensorRT version matches installed CUDA, TensorRT, and PyTorch.
- TensorRT 10 migration issues:
  Use the current `inference_tensorrt.py` in this repo, which handles the TensorRT 10 I/O tensor execution path.
